What is Continuous Summarization?

Software projects generate a constant stream of activity — commits, pull requests, issues, discussions, releases, CI runs. No single person can track it all. Continuous Summarization uses an AI agent on a schedule (daily, weekly, or per-event) to read all recent activity, distill it into a concise, actionable report, and publish it as a GitHub issue or discussion. Think of it as an automated standup that never misses a beat.

📰

Daily Digest Reports

Automatically generated summaries of the last 24 hours: new issues, merged PRs, CI status, release activity, and contributor highlights — delivered as a GitHub issue every morning.

📈

Trend Analysis

The agent tracks patterns over time: increasing bug reports in a module, slowing PR review velocity, or growing test failures — surfacing trends before they become problems.

🎯

Actionable Next Steps

Every summary ends with concrete recommendations: stale PRs that need review, issues that need triage, and blockers that need attention from specific team members.

🔄

Auto-Close Older Reports

Using the close-older-issues safe-output option, previous reports are automatically closed when a new one is created, keeping your issue tracker clean.

Example Workflow

A daily summarization workflow that creates a team status report every morning.

📄 .github/workflows/daily-summary.md 
---
name: Daily Project Summary
description: Generate a daily digest of repo activity

on:
  schedule:
    - cron: "0 9 * * 1-5"  # Weekdays 9am UTC
  workflow_dispatch:

permissions:
  contents: read
  issues: read
  pull-requests: read

safe-outputs:
  create-issue:
    title-prefix: "[daily-summary] "
    labels:
      - report
      - daily-summary
    close-older-issues: true

tools:
  github:
---

# Daily Project Summary

You are a project analyst for
${{ github.repository }}.

Create an upbeat daily status report covering:

1. Recent repository activity
   - Issues opened/closed in last 24h
   - PRs merged/opened/stalled
   - Commits and contributors
2. CI/CD health
   - Recent workflow run results
   - Any failures or flaky tests
3. Progress highlights
   - Notable achievements
   - Milestones approaching
4. Actionable recommendations
   - PRs needing review
   - Stale issues to close
   - Blockers to address

Format as a well-structured GitHub issue.
Use tables and lists for clarity.
Keep it concise and actionable.

Key Design Decisions

  • Schedule trigger: Runs on weekday mornings so the team has a fresh report at the start of each day
  • close-older-issues: Automatically closes yesterday's report when today's is created — no manual cleanup
  • Title prefix: [daily-summary] makes reports easy to filter and search
  • Read-only + single issue: The agent can only create one issue; it cannot modify code, merge PRs, or close existing issues
  • workflow_dispatch: Allows manual triggering for on-demand reports

Summarization Variants

Continuous Summarization adapts to different cadences and audiences.

📅

Weekly Rollup

A comprehensive weekly report with trend charts, velocity metrics, and sprint progress. Ideal for engineering managers and stakeholders who need the bigger picture.

🚀

Release Notes Generator

Triggered on release events, the agent reads all commits and PRs since the last tag and generates polished, user-facing release notes.

🔔

Event-Driven Summaries

Trigger on milestone completion, large PR merges, or security advisories to generate targeted summaries for specific audiences.
