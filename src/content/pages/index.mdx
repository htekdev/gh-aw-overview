<section id="what-is">
  <div class="container">
    <span class="section-label">Introduction</span>
    <h2 class="section-title">What is GitHub Agentic Workflows?</h2>
    <div class="what-is-grid">
      <div class="what-is-text">
        <p>
          <strong>GitHub Agentic Workflows (gh-aw)</strong> are AI-powered automations that can
          understand context, make decisions, and take meaningful actions — all from
          <strong>natural language instructions written in Markdown</strong>.
        </p>
        <p>
          Unlike traditional automation with fixed if-then rules, agentic workflows use
          <strong>coding agents</strong> to interpret your instructions and carry out tasks that
          normally require human judgment — triaging issues, investigating CI failures, updating
          documentation, and improving tests.
        </p>
        <p>
          Created by <strong>Don Syme</strong> and <strong>Peli de Halleux</strong> at
          <strong>GitHub Next</strong>, the project is fully open source under the MIT license,
          with 313+ releases and a vibrant community.
        </p>
        <div class="quote-block">
          <p>"A useful mental model: if repetitive work in a repository can be described in words,
             it might be a good fit for an agentic workflow."</p>
          <cite>— GitHub Blog</cite>
        </div>
      </div>
      <div class="what-is-visual">
        <h3 class="what-is-visual-heading">How It Works</h3>
        <div class="flow-step">
          <div class="flow-num">1</div>
          <div class="flow-content">
            <h4>Write</h4>
            <p>Describe automation intent in Markdown with YAML frontmatter</p>
          </div>
        </div>
        <div class="flow-step">
          <div class="flow-num">2</div>
          <div class="flow-content">
            <h4>Compile</h4>
            <p>Transform to a hardened Actions workflow via <code>gh aw compile</code></p>
          </div>
        </div>
        <div class="flow-step">
          <div class="flow-num">3</div>
          <div class="flow-content">
            <h4>Push</h4>
            <p>Commit both <code>.md</code> source and <code>.lock.yml</code> to your repo</p>
          </div>
        </div>
        <div class="flow-step">
          <div class="flow-num">4</div>
          <div class="flow-content">
            <h4>Run</h4>
            <p>Triggers automatically via GitHub Actions on events or schedules</p>
          </div>
        </div>
        <div class="flow-step">
          <div class="flow-num">5</div>
          <div class="flow-content">
            <h4>Review</h4>
            <p>Agent produces issues, PRs, and comments for human review</p>
          </div>
        </div>
      </div>
    </div>
  </div>

<section id="continuous-ai">
  <div class="container">
    <span class="section-label">The Vision</span>
    <h2 class="section-title">The Six Pillars of Continuous AI</h2>
    <p class="section-desc">
      Continuous AI is the systematic, automated application of AI to software collaboration —
      natural-language rules combined with agentic reasoning, executed continuously inside your repository.
      It augments CI/CD for judgment-heavy, context-dependent tasks.
    </p>
    <div class="pillars-grid">
      <div class="pillar-card">
        <div class="pillar-icon">🏷️</div>
        <h3>Continuous Triage</h3>
        <p>Automatically summarize, label, and route new issues. The agent understands context and intent — far beyond keyword matching.</p>
      </div>
      <div class="pillar-card">
        <div class="pillar-icon">📝</div>
        <h3>Continuous Documentation</h3>
        <p>Keep READMEs and docs aligned with code changes. Detect mismatches between docstrings and implementations, then open PRs to fix them.</p>
      </div>
      <div class="pillar-card">
        <div class="pillar-icon">🧹</div>
        <h3>Continuous Code Simplification</h3>
        <p>Identify refactoring opportunities — dead code, complex conditionals, repeated patterns — and propose targeted improvements via PRs.</p>
      </div>
      <div class="pillar-card">
        <div class="pillar-icon">🧪</div>
        <h3>Continuous Test Improvement</h3>
        <p>Assess test coverage and add high-value tests. One experiment: 5% → near 100% coverage, 1,400+ tests written over 45 days for ~$80.</p>
      </div>
      <div class="pillar-card">
        <div class="pillar-icon">🔧</div>
        <h3>Continuous Quality Hygiene</h3>
        <p>Proactively investigate CI failures and propose targeted fixes — flaky tests, dependency conflicts, and configuration issues.</p>
      </div>
      <div class="pillar-card">
        <div class="pillar-icon">📊</div>
        <h3>Continuous Reporting</h3>
        <p>Generate regular reports on repository health, synthesizing data from issues, PRs, commits, and CI into actionable insights.</p>
      </div>
    </div>
  </div>
</section>
