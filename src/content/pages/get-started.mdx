<section id="prerequisites">
<div class="section-label">Before You Begin</div>

## Prerequisites

<div class="card-grid-2col">
  <div class="info-card">
    <h3>✅ Required</h3>
    <ul>
      <li>A <strong>GitHub Copilot</strong> subscription (Individual, Business, or Enterprise)</li>
      <li><strong>GitHub CLI</strong> v2.45.0 or later</li>
      <li>The <code>gh-aw</code> CLI extension</li>
      <li><strong>Admin access</strong> to the target repository</li>
    </ul>
  </div>
  <div class="info-card">
    <h3>🔑 Tokens &amp; Secrets</h3>
    <ul>
      <li><code>COPILOT_GITHUB_TOKEN</code> — <strong>required</strong> for every workflow</li>
      <li><code>GH_AW_AGENT_TOKEN</code> — optional, needed for <code>assign-to-agent</code></li>
      <li>Custom <strong>PAT or GitHub App</strong> — optional, for cross-repo or elevated access</li>
    </ul>
  </div>
</div>
</section>

<section id="install">
<div class="section-label">Setup</div>

## Install &amp; Initialize

<p class="section-desc">Three commands to go from zero to a fully initialized repository. The CLI handles scaffolding, engine selection, and configuration.</p>

<div class="step-grid">
  <div class="step-item">
    <div class="step-num">1</div>
    <div class="step-body">
      <h4>Install the gh-aw CLI extension</h4>
      <p>Adds the <code>gh aw</code> command to your GitHub CLI. Requires GitHub CLI v2.45.0 or later.</p>
    </div>
  </div>
  <div class="step-item">
    <div class="step-num">2</div>
    <div class="step-body">
      <h4>Initialize your repository</h4>
      <p>Run <code>gh aw init --engine copilot</code> to scaffold your repo. This creates <code>copilot-setup-steps.yml</code>, walks you through engine selection, and configures the necessary directory structure.</p>
    </div>
  </div>
  <div class="step-item">
    <div class="step-num">3</div>
    <div class="step-body">
      <h4>Create the required secret</h4>
      <p>Store your Copilot token as a repository secret so workflows can authenticate with the Copilot API.</p>
    </div>
  </div>
</div>

<div class="code-block">
  <div class="code-header"><span>Terminal</span></div>
  <pre>{`# 1. Install the gh-aw CLI extension
gh extension install github/gh-aw

# 2. Initialize your repository
gh aw init --engine copilot

# 3. Create the required secret
gh secret set COPILOT_GITHUB_TOKEN -R owner/repo`}</pre>
</div>

<div class="callout">
  <strong>💡 What does <code>gh aw init</code> do?</strong> It creates the <code>copilot-setup-steps.yml</code> file that defines your agent's runtime environment, walks you through engine selection (Copilot, Claude, or Codex), and sets up the <code>.github/workflows/</code> directory for your first workflow.
</div>
</section>

<section id="token">
<div class="section-label">Authentication</div>

## Create the COPILOT_GITHUB_TOKEN

<p class="section-desc">Every agentic workflow needs a <code>COPILOT_GITHUB_TOKEN</code> to authenticate API calls. Here's how to create one:</p>

<div class="step-grid">
  <div class="step-item">
    <div class="step-num">1</div>
    <div class="step-body">
      <h4>Generate a fine-grained token</h4>
      <p>Go to <a href="https://github.com/settings/personal-access-tokens/new" target="_blank" rel="noopener noreferrer">github.com/settings/personal-access-tokens/new</a> and select <strong>Fine-grained personal access token</strong>.</p>
    </div>
  </div>
  <div class="step-item">
    <div class="step-num">2</div>
    <div class="step-body">
      <h4>Grant Copilot permission</h4>
      <p>Under <strong>Account permissions</strong>, set <strong>Copilot Requests</strong> to <strong>Read-only</strong>. No other permissions are required for the base token.</p>
    </div>
  </div>
  <div class="step-item">
    <div class="step-num">3</div>
    <div class="step-body">
      <h4>Store as a repository secret</h4>
      <p>Run <code>gh secret set COPILOT_GITHUB_TOKEN -R owner/repo</code> and paste the token when prompted — or add it via the repository Settings → Secrets → Actions page.</p>
    </div>
  </div>
</div>

<section id="first-workflow">
<div class="section-label">Build</div>

## Your First Workflow

<p class="section-desc">Use the CLI to add a pre-built workflow template, compile it into a hardened GitHub Actions file, and push it to your repo.</p>

<div class="step-grid">
  <div class="step-item">
    <div class="step-num">1</div>
    <div class="step-body">
      <h4>Add a workflow from the gallery</h4>
      <p>The <code>add-wizard</code> command pulls a workflow template and saves it as a Markdown source file in <code>.github/workflows/</code>.</p>
    </div>
  </div>
  <div class="step-item">
    <div class="step-num">2</div>
    <div class="step-body">
      <h4>Compile to a hardened Actions workflow</h4>
      <p>The <code>compile</code> command transforms your Markdown source into a <code>.lock.yml</code> file with pinned actions, sandboxed permissions, and safe outputs.</p>
    </div>
  </div>
  <div class="step-item">
    <div class="step-num">3</div>
    <div class="step-body">
      <h4>Push to your repository</h4>
      <p>Commit both the <code>.md</code> source and the <code>.lock.yml</code> compiled file. GitHub Actions picks up the lock file automatically on the next trigger.</p>
    </div>
  </div>
</div>

<div class="code-block">
  <div class="code-header"><span>Terminal</span></div>
  <pre>{`# 1. Create a workflow from a template
gh aw add-wizard githubnext/agentics/daily-repo-status

# 2. Compile to a hardened Actions workflow
gh aw compile .github/workflows/daily-repo-status.md

# 3. Push both files to your repo
git add .github/workflows/
git commit -m "Add daily status report workflow"
git push`}</pre>
</div>

<div class="callout">
  <strong>🎉 That's it!</strong> Your workflow will now trigger automatically based on its configured events (schedule, issue creation, etc.). Check the Actions tab in your repository to see it run.
</div>
</section>
