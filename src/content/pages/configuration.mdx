### What `gh aw init` Does

Running `gh aw init` in your repository:

<div class="card card-accent-green" style={{marginBottom: '16px'}}>
  <ul style={{margin: '8px 0 0', paddingLeft: '20px', lineHeight: '2'}}>
    <li>Ō£ģ Creates <code>.github/workflows/copilot-setup-steps.yml</code> ŌĆö the environment setup file for the agentic workflow runner</li>
    <li>Ō£ģ Walks you through selecting an AI engine (Copilot, Claude, Codex)</li>
    <li>Ō£ģ Provides instructions for setting up required secrets</li>
    <li>ŌØī Does <strong>not</strong> create the <code>COPILOT_GITHUB_TOKEN</code> secret ŌĆö you must do this manually</li>
  </ul>
</div>

```bash
# Initialize the repo
gh aw init --engine copilot

# Then manually create the required secret
gh secret set COPILOT_GITHUB_TOKEN -R owner/repo
# (paste your fine-grained PAT when prompted)
```

<div class="callout callout-orange">
  <p>ŌÜĀ’ĖÅ Without <code>COPILOT_GITHUB_TOKEN</code>, the activation step will fail immediately. This is the first thing to check when troubleshooting a failed workflow run.</p>

<div class="section-hdr" id="overview">
  <h2>How Configuration Works</h2>
</div>

Every agentic workflow is a `.md` file in `.github/workflows/` with two parts:

<div class="card-grid-2col">
  <div class="card card-accent-blue">
    <h3>­¤ōŗ YAML Frontmatter</h3>
    <p class="desc">Structured config between <code>---</code> markers ŌĆö triggers, permissions, tools, engine, and safe outputs. Validated at compile time.</p>
  </div>
  <div class="card card-accent-green">
    <h3>­¤ōØ Markdown Body</h3>
    <p class="desc">Natural-language instructions for the AI agent. Editable without recompilation ŌĆö the agent interprets these contextually at runtime.</p>
  </div>
</div>

```yaml
# File: .github/workflows/triage-issues.md
---
on:
  issues:
    types: [opened]
permissions: read-all
safe-outputs:
  add-labels:
    allowed: [bug, feature, docs]
  add-comment: {}
---

# Issue Triage Agent
Analyze new issues and apply the appropriate label.
Explain your reasoning in a comment.
```

<div class="callout callout-green">
  <p>­¤ÆĪ Run <code>gh aw compile</code> to transform the <code>.md</code> file into a <code>.lock.yml</code> ŌĆö a security-hardened GitHub Actions workflow. The lock file is what Actions executes.</p>
</div>

<div class="section-hdr" id="auth">
  <h2>Authentication & Tokens</h2>
</div>

Agentic workflows use repository secrets for authentication. The compiled `.lock.yml` workflow references these secrets at runtime during the activation and safe-outputs steps.

### `COPILOT_GITHUB_TOKEN` ŌĆö The Core Requirement

Every agentic workflow needs `COPILOT_GITHUB_TOKEN` ŌĆö a **fine-grained PAT** from a user with an active Copilot license. The compiled workflow validates this secret during activation; without it, the run fails immediately.

<div class="card card-accent-blue" style={{marginBottom: '24px'}}>
  <h3>Creating the Token</h3>
  <ol style={{margin: '12px 0 0', paddingLeft: '20px', lineHeight: '2'}}>
    <li>Go to <a href="https://github.com/settings/personal-access-tokens/new" target="_blank" rel="noopener noreferrer">github.com/settings/personal-access-tokens</a></li>
    <li>Select <strong>Fine-grained token</strong> ŌåÆ choose your target repository or <em>Public Repositories</em></li>
    <li>Under <strong>Account permissions</strong>, grant <strong>Copilot Requests: Read-only</strong></li>
    <li>Copy the generated token and store it as a repo secret:</li>
  </ol>
</div>

```bash
gh secret set COPILOT_GITHUB_TOKEN -R owner/repo
# Paste your fine-grained PAT when prompted
```

<div class="callout callout-orange">
  <p>ŌÜĀ’ĖÅ <code>COPILOT_GITHUB_TOKEN</code> is <strong>not</strong> the built-in <code>GITHUB_TOKEN</code>. It must be a PAT from a user whose Copilot subscription is consumed for premium requests. The token owner's account must have Copilot access.</p>
</div>

### Additional Secrets

Depending on your workflow configuration, you may need additional repository secrets:

| Secret | When Needed | Permission |
|---|---|---|
| `COPILOT_GITHUB_TOKEN` | **Always** ŌĆö every agentic workflow | Copilot Requests: Read-only |
| `GH_AW_AGENT_TOKEN` | Using `assign-to-agent` safe output | Issues: Read & Write |
| `ANTHROPIC_API_KEY` | Using `engine: claude` | ŌĆö |
| `OPENAI_API_KEY` | Using `engine: codex` | ŌĆö |
| `GEMINI_API_KEY` | Using `engine: gemini` | ŌĆö |
| `GH_AW_SECURITY_ALERTS_TOKEN` | Security monitoring workflows | Dependabot, Code scanning, Secret scanning alerts |

<div class="callout callout-blue">
  <p>­¤ÆĪ All secrets are stored as <strong>regular repository secrets</strong> (Settings ŌåÆ Secrets and variables ŌåÆ Actions). MCP server secrets referenced in the workflow frontmatter also go here, using the <code>{'${{ secrets.NAME }}'}</code> syntax.</p>
</div>

### Custom Tokens for Safe Outputs

By default, safe-output write operations use the built-in `GITHUB_TOKEN`. To use a **different identity** for writes (e.g., a bot account, a GitHub App), configure `github-token:` or `github-app:` in your safe-outputs.

<div class="card-grid-2col">
  <div class="card card-accent-blue">
    <h3>­¤öæ Fine-Grained PAT</h3>
    <p class="desc">Use a PAT for full control. Supports all safe-output types including <code>assign-to-agent</code>.</p>

```yaml
safe-outputs:
  github-token:
    secret: MY_BOT_PAT
  create-pull-request:
    draft: true
  assign-to-agent:
    allowed: [copilot]
```

  </div>
  <div class="card card-accent-orange">
    <h3>­¤ż¢ GitHub App</h3>
    <p class="desc">Use a GitHub App for scoped, short-lived tokens. Better for orgs ŌĆö no personal account needed.</p>

```yaml
safe-outputs:
  github-app:
    app-id: ${{ vars.APP_ID }}
    private-key: ${{ secrets.APP_KEY }}
  create-pull-request:
    draft: true
  add-comment: {}
```

  </div>
</div>

#### PAT Required Scopes

| Scope | Required For |
|---|---|
| `contents: write` | Creating PRs, pushing code |
| `issues: write` | Creating/updating issues, adding labels |
| `pull-requests: write` | PR reviews, comments |
| `actions: write` | Dispatching workflows |

<div class="callout callout-orange">
  <p>ŌÜĀ’ĖÅ <strong>GitHub App limitation:</strong> The <code>assign-to-agent</code> safe output requires a <strong>fine-grained PAT</strong> ŌĆö GitHub App installation tokens are not supported by the Copilot assignment API. If you use <code>github-app:</code> for other safe outputs, set the <code>GH_AW_AGENT_TOKEN</code> repository secret (a PAT) as a fallback for agent assignment.</p>
</div>

### Token Summary

| Scenario | Token Type | Where to Configure |
|---|---|---|
| Running any agentic workflow | Fine-grained PAT | `COPILOT_GITHUB_TOKEN` repo secret |
| Third-party engine (Claude, Codex, Gemini) | API key | Engine-specific repo secret |
| MCP server authentication | Secret | Repo secret, referenced via `${{ secrets.NAME }}` |
| Safe-output writes (custom identity) | PAT or GitHub App | `github-token:` or `github-app:` in frontmatter |
| Assign-to-agent | Fine-grained PAT only | `GH_AW_AGENT_TOKEN` repo secret |

### Complete Workflow Example

```yaml
---
name: "CI Failure Investigator"
description: "Investigates CI failures and proposes fixes"

on:
  workflow_run:
    workflows: ["CI"]
    types: [completed]
  roles: [admin, maintainer]
  stop-after: "+90d"

permissions:
  contents: read
  actions: read
  issues: read

engine:
  id: copilot
  max-continuations: 3

tools:
  edit:
  bash: ["gh", "git:*", "npm"]
  github:
    toolsets: [repos, issues, pull_requests]

safe-outputs:
  create-pull-request:
    title-prefix: "[ci-fix] "
    labels: [ci-fix, automation]
    draft: true
  add-comment:
    max: 2

network:
  allowed:
    - defaults
    - node
    - github

timeout-minutes: 15
---

# CI Failure Investigator

When a CI workflow fails, investigate the failure:

1. Read the failed workflow logs
2. Identify the root cause
3. Search the codebase for the relevant code
4. Propose a fix as a pull request
5. Comment on the triggering issue with your analysis
```
