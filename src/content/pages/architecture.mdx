<section id="compilation">
<div class="section-label" style="color: var(--color-purple);">Compilation</div>

## Four-Phase Compilation Pipeline

<p class="section-desc">The <code>gh aw compile</code> command transforms your Markdown workflow through four sequential security phases before producing the <code>.lock.yml</code> execution plan.</p>

<div class="compile-steps">
  <div class="compile-step">
    <div class="compile-step-num">1</div>
    <h4>Schema Validation</h4>
    <p>Every frontmatter field is validated against a JSON schema. Invalid field names, wrong types, and malformed configurations are rejected before any code is generated.</p>
    <span class="mono-sm">Invalid fields → rejected</span>
  </div>
  <div class="compile-step">
    <div class="compile-step-num">2</div>
    <h4>Expression Safety</h4>
    <p>Only allowlisted GitHub Actions expressions are permitted. Expressions referencing <code>{'secrets.*'}</code> in unsafe positions are blocked — preventing injection attacks.</p>
    <span class="mono-sm">No secrets.{'*'} in unsafe positions</span>
  </div>
  <div class="compile-step">
    <div class="compile-step-num">3</div>
    <h4>Action SHA Pinning</h4>
    <p>Every Action reference is resolved to its full commit SHA and recorded in <code>actions-lock.json</code>. Defends against tag-hijacking and supply-chain attacks.</p>
    <span class="mono-sm">actions/checkout@v4 → SHA</span>
  </div>
  <div class="compile-step">
    <div class="compile-step-num">4</div>
    <h4>Security Scanners</h4>
    <p>The compiled output passes through <strong>actionlint</strong> (workflow linting), <strong>zizmor</strong> (vulnerability detection), and <strong>poutine</strong> (supply-chain risk analysis).</p>
    <span class="mono-sm">--actionlint --zizmor --poutine</span>
  </div>
</div>

<div class="grid-3" style="margin-top: 32px;">
  <div class="card">
    <span class="badge badge-blue">Linting</span>
    <h4 style="margin-bottom: 8px;">actionlint</h4>
    <p style="font-size: 0.88rem; color: var(--color-text-muted);">Workflow structure linting with integrated <strong>shellcheck</strong> and <strong>pyflakes</strong>. Catches syntax errors, type mismatches, and workflow misconfigurations.</p>
  </div>
  <div class="card">
    <span class="badge badge-red">Security</span>
    <h4 style="margin-bottom: 8px;">zizmor</h4>
    <p style="font-size: 0.88rem; color: var(--color-text-muted);">Security vulnerability detection. Scans for privilege escalation paths, security misconfigurations, and dangerous patterns in the compiled workflow.</p>
  </div>
  <div class="card">
    <span class="badge badge-orange">Supply Chain</span>
    <h4 style="margin-bottom: 8px;">poutine</h4>
    <p style="font-size: 0.88rem; color: var(--color-text-muted);">Supply-chain risk analysis. Evaluates third-party action risks, dependency vulnerabilities, and potential for compromised upstream actions.</p>

## Three-Layer Trust Model

<p class="section-desc">Each layer enforces distinct security properties under different failure assumptions. Breaching one layer does not compromise the guarantees of layers below it.</p>

<div class="trust-stack">
  <div class="trust-layer trust-layer-top">
    <div class="trust-layer-label">Layer 3 — Plan-Level Trust</div>
    <h4>Staged Execution &amp; Safe Outputs</h4>
    <p>Constrains component behavior <em>over time</em>. Decomposes workflows into stages with distinct active components, permissions, and admissible data consumers. The SafeOutputs subsystem is the primary instantiation.</p>
    <div class="trust-chips">
      <span class="trust-chip">Pre-activation checks</span>
      <span class="trust-chip">Content sanitization</span>
      <span class="trust-chip">Agent execution (read-only)</span>
      <span class="trust-chip">Secret redaction</span>
      <span class="trust-chip">Threat detection</span>
      <span class="trust-chip">Safe output application</span>
      <span class="trust-chip">Temporal constraints</span>
    </div>
  </div>
  <div class="trust-layer trust-layer-mid">
    <div class="trust-layer-label">Layer 2 — Configuration-Level Trust</div>
    <h4>Compiler, Schemas &amp; SHA Pinning</h4>
    <p>Declarative artifacts and toolchains that instantiate the system's structure. Controls which components are loaded, how they connect, which channels are permitted, and what privileges each receives. Token distribution is controlled here.</p>
    <div class="trust-chips">
      <span class="trust-chip">JSON Schema validation</span>
      <span class="trust-chip">Expression allowlist</span>
      <span class="trust-chip">SHA pinning</span>
      <span class="trust-chip">Security scanners</span>
      <span class="trust-chip">Strict mode</span>
      <span class="trust-chip">Token distribution</span>
    </div>
  </div>
  <div class="trust-layer trust-layer-bot">
    <div class="trust-layer-label">Layer 1 — Substrate-Level Trust</div>
    <h4>VM, Docker, iptables &amp; Hardware</h4>
    <p>Hardware and OS foundation. Provides memory isolation, CPU/resource isolation, and kernel-enforced communication boundaries. Guarantees hold even if the agent container is fully compromised.</p>
    <div class="trust-chips">
      <span class="trust-chip">CPU / MMU / Kernel</span>
      <span class="trust-chip">Hypervisor</span>
      <span class="trust-chip">Docker runtime</span>
      <span class="trust-chip">Network Firewall (AWF)</span>

      <span class="trust-chip">iptables</span>
    </div>
  </div>
</div>

<div class="callout">

</div>

<p class="section-desc">The agent runs inside the AWF's containerized environment with controlled filesystem access, network isolation, and engine selection.</p>

<div class="grid-2">

    <span class="badge badge-blue">Filesystem</span>
    <h4 style="margin-bottom: 12px;">Chroot Mode</h4>
    <p style="font-size: 0.88rem; color: var(--color-text-muted); margin-bottom: 16px;">Many coding agents expect full host access. The AWF provides a chroot mode that gives the agent access to host-installed tools while maintaining isolation:</p>
    <ul class="container-card-list">

      <li><span class="list-icon" style="color: var(--color-blue);">3</span> Mounts <code>HOME</code> and <code>/tmp</code> <strong>read-write</strong></li>
      <li><span class="list-icon" style="color: var(--color-blue);">4</span> Imports subset of host env vars (<code>USER</code>, <code>PATH</code>)</li>
      <li><span class="list-icon" style="color: var(--color-blue);">5</span> Launches agent in a <code>/host</code> chroot jail</li>
    </ul>
